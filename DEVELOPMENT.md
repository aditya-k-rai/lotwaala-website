# Lotwaala — Development Guide

## Overview

Lotwaala is a **high-performance SEO landing hub** for a wholesale marketplace mobile app. The website does **not** include e-commerce functionality. Its purpose is to:

1. **Rank for high-value keywords** — "Wholesaler App", "Wholesale Marketplace", "Bulk Inventory India"
2. **Drive app downloads** — every page funnels users toward Android/iOS download
3. **Capture long-tail search traffic** — via 240+ programmatic SEO landing pages

---

## Architecture

### Why Next.js App Router?

| Concern | App Router Advantage |
|---------|---------------------|
| **Speed** | Static Generation (SSG) via `generateStaticParams` — pages are pre-rendered at build time, served from CDN with zero server compute |
| **SEO** | `generateMetadata` API generates unique `<title>`, `<meta description>`, and Open Graph tags per page — critical for programmatic SEO |
| **Structured Data** | JSON-LD scripts are injected server-side, so crawlers see them on first paint |
| **Performance** | No client-side JS is needed for SEO pages — pure HTML/CSS served to crawlers |
| **Scalability** | Adding new cities or categories is a single array push — the build system generates all route permutations automatically |

### Tech Stack

- **Framework:** Next.js 16 (App Router, React 19, Turbopack)
- **Language:** TypeScript (strict mode)
- **Styling:** Tailwind CSS v4
- **Icons:** Lucide React
- **Deployment target:** Vercel / any static host (all pages are SSG)

## SEO Strategy

### 1. Programmatic SEO Footprint

The core SEO strategy uses **City + Category permutations** to capture long-tail search traffic:

```
/market/delhi/electronics       → "Wholesale Electronics in Delhi"
/market/mumbai/clothing         → "Wholesale Clothing in Mumbai"
/market/bangalore/cosmetics     → "Wholesale Cosmetics in Bangalore"
```

**Current scale:** 20 cities × 12 categories = **240 unique landing pages**

Each page gets:
- **Unique `<title>`** — e.g., `Wholesale Electronics in Delhi | Lotwaala — Wholesaler App`
- **Unique `<meta description>`** — mentions the city, category, app name, and CTA
- **Unique H1** — `Wholesale {Category} in {City}`
- **JSON-LD schema** — `WebPage` + `SoftwareApplication` for rich results
- **Cross-links** — links to other categories in the same city + same category in other cities

This interlinking creates a **dense internal link graph** that distributes page authority.

### 2. `SoftwareApplication` Schema

The home page and every market page include JSON-LD `SoftwareApplication` schema:

```json
{
  "@type": "SoftwareApplication",
  "name": "Lotwaala — Wholesale Marketplace",
  "operatingSystem": "Android, iOS",
  "applicationCategory": "BusinessApplication",
  "offers": { "@type": "Offer", "price": "0", "priceCurrency": "INR" },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.8",
    "ratingCount": "12400"
  }
}
```

This tells Google the site represents a **downloadable app**, enabling:
- Star ratings in search results
- "Free" pricing display
- App download buttons in SERPs

### 3. Metadata API

All metadata is generated by `generatePageMetadata()` in `lib/seo.ts`. It:
- Builds a keyword-rich `<title>` with the pattern: `{Page Title} | Lotwaala — Wholesaler App`
- Creates a unique `<meta description>` per page
- Sets `canonical` URLs to prevent duplicate content
- Includes `openGraph` and `twitter` cards for social sharing
- Adds `appleWebApp` and `mobile-web-app-capable` meta tags targeting the "Wholesaler App" keyword

### 4. Sitemap & Robots

**`sitemap.ts`** auto-generates entries for:
- Home page (priority 1.0)
- All 240 market pages (priority 0.8)

**`robots.ts`** allows:
- `/` and `/market/` (all public pages)

And blocks:
- `/admin/`, `/account/`, `/api/`, `/dashboard/`, `/settings/`

## Component Architecture

### JsonLd Component

A thin wrapper that injects JSON-LD into the `<head>`:

```tsx
<JsonLd data={generateAppSchema()} />
```

Used on every page to inject the appropriate schema (App, WebSite, Organization, LocalMarket).

### PhoneMockup

A **pure-CSS phone mockup** built entirely with Tailwind — no images required. It renders a realistic app interface showing:
- Search bar, location pill, category chips
- Product listing cards with pricing
- Bottom navigation

This gives the site an "app showcase" feel without needing actual screenshots.

### Programmatic SEO Page (`/market/[city]/[category]/page.tsx`)

Each page:
1. Validates `city` and `category` params against the constants arrays
2. Returns `notFound()` for invalid combos
3. Generates unique metadata via `generateMetadata()`
4. Renders keyword-rich content with city/category interpolation
5. Cross-links to related pages (other cities for same category, other categories in same city)

## Performance

### Core Web Vitals Strategy

| Metric | Target | How |
|--------|--------|-----|
| **LCP** | < 2.5s | SSG (no server render), `next/font` with `display: swap`, no hero images (CSS-only mockup) |
| **FID/INP** | < 100ms | Minimal client JS — pages are mostly static HTML |
| **CLS** | < 0.1 | Fixed-dimension layout, font preloading, no layout shifts |

### Optimizations

- **Static Generation** — all 241 pages are pre-rendered at build time
- **Font optimization** — Inter loaded via `next/font/google` with `swap` display
- **No external images** — phone mockup is pure CSS/HTML
- **Minimal JS** — no client components needed for the landing pages
- **Tailwind v4** — zero-runtime CSS with tree shaking

## PWA (Progressive Web App)

The `manifest.ts` file makes the site installable on mobile devices:
- App name: "Lotwaala"
- Theme color: `#2563eb` (brand blue)
- Display: `standalone` (no browser chrome)
- Icons: 192px and 512px PNG (place in `public/icons/`)

To complete PWA setup, add a service worker for offline caching (e.g., using `next-pwa` or Workbox).

## Future Scaling

### Adding More Cities

Edit `lib/constants.ts` → add to the `CITIES` array:

```ts
{ slug: "varanasi", name: "Varanasi", state: "Uttar Pradesh" },
```

The build system automatically generates new `/market/varanasi/*` pages, updates the sitemap, and creates cross-links.

### Adding More Categories

Same approach — add to the `CATEGORIES` array:

```ts
{ slug: "hardware", name: "Hardware & Tools", description: "Power tools, hand tools & hardware" },
```

### Adding a Blog

Create `app/blog/[slug]/page.tsx` with:
- A headless CMS (Sanity, Contentful) or local MDX files
- `generateStaticParams` to pre-render all posts
- Target long-tail queries like:
  - "How to find wholesale electronics dealers in Delhi"
  - "How to liquidate dead stock in Mumbai"
  - "Best wholesale markets in India 2026"
  - "Wholesale clothing business tips for beginners"

Add blog URLs to `sitemap.ts` for indexing.

### Adding E-Commerce

If the project evolves to include transactions:
1. Add `app/lot/[id]/page.tsx` for individual listing pages
2. Add `Product` JSON-LD schema to listing pages
3. Switch from SSG to ISR (Incremental Static Regeneration) for dynamic inventory
4. Add authentication routes under `/account/` (already blocked in robots.txt)
